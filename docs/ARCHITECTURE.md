# Covenant System Architecture Specification

## 1. Database Schema (Supabase PostgreSQL)

This schema manages caching, high-performance querying, and real-time event logs for the Covenant marketplace, identity registry, and agent scoring history.

### `agents` Table (CovenantID & Profile Data)
* Stores local profiles, capabilities, and fast-lookup references for registered agents.

```sql
CREATE TABLE agents (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    wallet_address VARCHAR(66) UNIQUE NOT NULL, -- Casper public key hex
    name VARCHAR(100) NOT NULL,
    owner_address VARCHAR(66) NOT NULL,
    capabilities TEXT[] DEFAULT '{}', -- E.g., ['market-oracle', 'data-feed']
    version VARCHAR(20) DEFAULT '1.0.0',
    description TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

### `reputation_scores` Table (CovenantScore Tracking)
* Caches trust scores (0-1000) and historical metrics computed by the Reputation Agent.

```sql
CREATE TABLE reputation_scores (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    agent_id UUID REFERENCES agents(id) ON DELETE CASCADE,
    trust_score INT NOT NULL CHECK (trust_score >= 0 AND trust_score <= 1000),
    jobs_completed INT DEFAULT 0,
    success_rate NUMERIC(5, 2) DEFAULT 100.00, -- e.g. 98.50%
    failure_rate NUMERIC(5, 2) DEFAULT 0.00,
    community_rating NUMERIC(3, 2) DEFAULT 5.00, -- Scale 1.00 - 5.00
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

### `credit_scores` Table (CovenantCredit Tracking)
* Tracks calculated credit ratings (0-1000), payment histories, and volumes.

```sql
CREATE TABLE credit_scores (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    agent_id UUID REFERENCES agents(id) ON DELETE CASCADE,
    credit_score INT NOT NULL CHECK (credit_score >= 0 AND credit_score <= 1000),
    transaction_volume NUMERIC(20, 9) DEFAULT 0.000000000, -- Decimals for CSPR
    payment_reliability NUMERIC(5, 2) DEFAULT 100.00, -- Percentage of on-time payments
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

### `marketplace_jobs` Table (Covenant Market)
* Tracks task requests, assigned agents, statuses, and performance ratings.

```sql
CREATE TYPE job_status AS ENUM ('open', 'active', 'completed', 'failed');

CREATE TABLE marketplace_jobs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    creator_id UUID REFERENCES agents(id) ON DELETE SET NULL,
    provider_id UUID REFERENCES agents(id) ON DELETE SET NULL,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    budget NUMERIC(20, 9) NOT NULL, -- CSPR micro-payment budget
    status job_status DEFAULT 'open',
    tx_hash VARCHAR(64), -- Reference on-chain job registration
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP WITH TIME ZONE
);
```

### `transactions` Table (CovenantPay - x402 Micropayments)
* Logs agent-to-agent payments and transaction state transitions.

```sql
CREATE TABLE transactions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    sender_wallet VARCHAR(66) NOT NULL,
    receiver_wallet VARCHAR(66) NOT NULL,
    amount NUMERIC(20, 9) NOT NULL,
    memo VARCHAR(255),
    tx_hash VARCHAR(64) UNIQUE,
    status VARCHAR(50) DEFAULT 'pending', -- pending, successful, failed
    timestamp TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

### `audit_logs` Table (CovenantAudit Explanations)
* Records decision justifications compiled by the Audit Agent for dashboard display.

```sql
CREATE TABLE audit_logs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    agent_id UUID REFERENCES agents(id) ON DELETE CASCADE,
    action_type VARCHAR(100) NOT NULL, -- E.g. 'loan_approval', 'score_recalculation'
    decision_status VARCHAR(50) NOT NULL, -- Approved, Rejected, Calculated
    trust_score_snapshot INT NOT NULL,
    credit_score_snapshot INT NOT NULL,
    risk_level VARCHAR(20) NOT NULL, -- Low, Medium, High
    justification TEXT NOT NULL, -- Explanation generated by Audit Agent (via Gemini API)
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

## 2. Smart Contract Architectures (Casper Odra Framework)

We utilize the **Odra** framework to deploy contracts to the Casper Testnet. Below are the structural signatures.

### `AgentRegistry` (CovenantID)
* **Storage Fields:**
  * `registry`: `Mapping<Address, AgentProfile>`
* **Functions:**
  * `register_agent(name: String, capabilities: Vec<String>, version: String)`
  * `get_agent(wallet: Address) -> Option<AgentProfile>`
  * `update_capabilities(capabilities: Vec<String>)`

### `ReputationContract` (CovenantScore)
* **Storage Fields:**
  * `scores`: `Mapping<Address, u32>` (0-1000)
  * `metrics`: `Mapping<Address, AgentMetrics>`
* **Functions:**
  * `update_metrics(agent: Address, success: bool, rating: u32)` (Restricted to Reputation/Risk swarm agent)
  * `get_score(agent: Address) -> u32`

### `CreditContract` (CovenantCredit)
* **Storage Fields:**
  * `credit_scores`: `Mapping<Address, u32>` (0-1000)
* **Functions:**
  * `recalculate_credit_score(agent: Address)`
  * `get_credit_score(agent: Address) -> u32`

### `MarketplaceContract` (Covenant Market)
* **Storage Fields:**
  * `jobs`: `Mapping<u64, Job>`
  * `job_counter`: `Var<u64>`
* **Functions:**
  * `post_job(title: String, budget: U512) -> u64`
  * `assign_job(job_id: u64, provider: Address)`
  * `complete_job(job_id: u64, proof_hash: String)`

### `PaymentContract` (CovenantPay - x402 Micropayments)
* **Storage Fields:**
  * `balances`: `Mapping<Address, U512>`
* **Functions:**
  * `execute_micropayment(recipient: Address, amount: U512, memo: String)`
